# OTPAP v2 Specification

Version: 2.0 Draft

## Abstract

OTPAP (One Time Password Application Protocol) is a stateful protocol designed to secure API invocations by binding each request to a specific execution context.

The protocol ensures that:

- a request can be executed only once;
- requests are bound to a session;
- requests are bound to a specific page context;
- requests are bound to a specific API endpoint;
- requests cannot be modified after token generation.

---

# Goals

The protocol provides protection against:

- Replay Attacks
- Request Forgery
- Parameter Manipulation
- Unauthorized API Invocation
- Context Switching Attacks

---

# Design Principles

OTPAP is based on the following principles:

1. Stateful validation.
2. One-time execution.
3. Strong context binding.
4. Request integrity.
5. Minimal cryptographic footprint.

---

# Core Components

## Session

Authenticated user context.

## Page

Logical application page generated by the server.

## OTPAP Token

Single-use authorization token.

## OTPAP Engine

Validation engine responsible for:

- token verification;
- replay detection;
- sequence validation;
- token consumption.
