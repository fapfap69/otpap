# Security Policy

OTPAP is a security protocol project, so responsible disclosure matters.

## Supported Versions

- `v2.0.0-draft` and future tagged releases are supported for security review.

## Reporting a Vulnerability

If you find a weakness in the specification, reference implementation, or examples, please report it privately to the maintainers before public disclosure.

Include:

- A short summary of the issue
- Impact analysis
- Affected files or protocol sections
- A reproducible proof of concept when possible
- Suggested remediation if you have one

## Response Expectations

The maintainers SHOULD acknowledge reports promptly, triage severity, and coordinate a fix or mitigation before public disclosure.

## Safe Testing

Security testing MUST be limited to systems you own or are explicitly authorized to test. Replay attacks, tampering tests, and load tests SHOULD be performed against isolated development environments.
